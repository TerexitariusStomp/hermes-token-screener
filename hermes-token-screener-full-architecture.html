Hermes Token Screener Full Architecture

Repository-grounded expansion of the README architecture: discovery → async enrichment → cross/social intelligence → copytrade feedback → multi-chain trading → dashboard/export/CI runtime.

Codebase Coverage

  • • Discovery scripts: telegram_scraper, token_discovery*, gmgn/pumpportal harvesters, integration/lifecycle.
  • • Enrichment stack: token_enricher + async_enrichment fan-out across required and optional layers.
  • • Ranking stack: wallet_tracker + async_wallets + cross_scoring/enhanced_scoring + social_enhancement.
  • • Intelligence modules: copytrade monitor, keyword discovery, website intelligence, Twitter/combined analyzers.
  • • Core package modules: config/logging/metrics, agents router/registry, models, memory, prompt tools.

State + Feedback Loops

  • • Primary data stores: central_contracts.db, wallet_tracker.db, copytrade_discoveries.db.
  • • Artifact bus: top100.json and phase snapshots (phase1_initial, phase3_smartmoney, phase4_social).
  • • Trading state: active positions, AI decisions, monitor logs, market history.
  • • Copytrade loop reinjects newly observed tokens/wallets back into discovery/enrichment.
  • • Social/keyword/website intelligence feeds final reranking before delivery and trading.

Delivery, Runtime, and Ops

  • • Live app: FastAPI dashboard (tokens, wallets, cross views, chart APIs).
  • • Static export: export_github_pages.py writes docs/data JSON for GitHub Pages.
  • • CI gates: ruff/black/pytest/mypy via .github/workflows/ci.yml.
  • • Deploy automation: pages.yml publishes docs/ on main push.
  • • Container + scheduling: Dockerfile, docker-compose, and cron-based script orchestration.
